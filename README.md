# Website JS Scanner

A modular, extensible web scanner for detecting malicious JavaScript and external links.

## 📖 Introduction

An automated website JavaScript file scanning and downloading tool. Enter a website domain, and the tool automatically crawls all pages, extracts and downloads all referenced JS files, while identifying external links and potential malicious links for offline security auditing and code analysis.

**Key Features:**
- Supports Selenium dynamic rendering to capture malicious links and scripts generated by Vue/React/AJAX
- Intelligent URL pattern deduplication - only scans one representative page per URL pattern
- Filename-based deduplication - downloads files with different names even if content is identical

## 🎯 Problems Solved

### 1. Malicious JS File Detection
- Detect if the website references known malicious scripts (e.g., bshare.cn and other compromised third-party plugins)
- Discover suspicious external JS links hidden in pages
- Audit third-party JS files for security vulnerabilities
- Download all JS files locally for offline analysis

### 2. Black Links and External Link Discovery
- Automatically identify external links in pages (excluding target domain and all subdomains)
- Supports extraction from 16+ tag types: `a`, `link`, `iframe`, `img`, `script`, `meta`, and more
- Detects event attributes: `onclick`, `onload`, `onmouseover` events containing malicious redirects
- Detects dynamic injection: `document.write` and `innerHTML` injected iframe/a tags
- Detects location redirects: `window.location` and `top.location` redirects
- Complete tag code recording for rapid localization

### 3. Dynamic Rendering Page Support
- **Vue/React Single Page Applications**: Selenium renders to obtain complete DOM
- **AJAX Asynchronous Loading**: Wait for async requests to complete before extraction
- **setTimeout Delayed Injection**: Wait for dynamic content to finish loading
- **Lazy-loaded Images/Links**: Scroll pages to trigger lazy loading

### 4. Security Incident Emergency Response
Quickly locate problem files when websites experience abnormal redirects, SEO black links, pop-up ads, etc.

### 5. Offline Auditing
Download all JS files locally for in-depth code analysis using grep, IDE, and other tools.

## ✨ Features

### Dynamic Rendering Engine
- ✅ **Selenium Dynamic Rendering** - Supports Vue/React/AJAX pages, runs in headless mode
- ✅ **Event Attribute Extraction** - Capture malicious URLs in onclick/onload/onmouseover
- ✅ **Dynamic Injection Detection** - Identify document.write/innerHTML injected black links
- ✅ **Location Redirect Detection** - Capture window.location/top.location malicious redirects
- ✅ **Lazy Loading Support** - Scroll pages to trigger image/link lazy loading
- ✅ **Alert Handling** - Automatically dismiss unexpected browser alerts (e.g., Baidu Maps license warnings)

### Core Features
- ✅ **Full-site Recursive Crawling** - Automatically crawl all pages under the same domain and all subdomains
- ✅ **Pattern-Based Deduplication** - Only scan one representative URL per pattern (e.g., `/products/1`, `/products/2` → scan only once)
- ✅ **Automatic JS Download** - Save all JS files locally
- ✅ **Filename-Based Deduplication** - Download files with different names even if content is identical
- ✅ **Filename Conflict Handling** - Auto-add counter when same filename already exists
- ✅ **Fresh Scan Each Time** - Auto-clear old directory, no residual files
- ✅ **Complete URL Recording** - JS file list includes full URLs for traceability
- ✅ **External Link Identification** - Automatically extract external links and record complete tag code
- ✅ **16+ Tag Support** - Covers `a`, `link`, `iframe`, `img`, `script`, `meta`, and more
- ✅ **Subdomain Support** - Subdomains are treated as internal links (not listed as external)
- ✅ **Error Handling** - Timeouts and connection errors auto-skipped without interrupting scan progress
- ✅ **Connectivity Check** - Verifies domain reachability before scanning; aborts gracefully if unreachable

## 📋 Supported Detection Scenarios

| Scenario | Solution | Example |
|----------|----------|---------|
| **Vue/React Dynamic Rendering** | Selenium renders and gets page_source | `<div id="app">{{ link }}</div>` |
| **AJAX Async Black Links** | WebDriverWait + delays | `setTimeout(() => { createLink() }, 3000)` |
| **onclick Malicious Redirect** | Iterate all event attributes | `<div onclick="open('https://evil.com')">` |
| **window.location Hijacking** | JS execution + regex matching | `window.location.href = 'https://evil.com'` |
| **document.write Injection** | Execute JS to detect injected content | `document.write('<iframe src="evil.com">')` |
| **innerHTML Black Link Injection** | Detect innerHTML assignment | `div.innerHTML = '<a href="evil.com">'` |
| **CSS Pseudo-element Hidden Links** | Selenium detects hidden elements | `style="display:none"`, `opacity:0` |
| **Zero-size Pixel Links** | Detect element dimension attributes | `width="0" height="0"` |
| **Dynamically Created a/iframe** | Selenium gets all DOM elements after rendering | `document.createElement('iframe')` |
| **Baidu Maps License Alerts** | Auto-dismiss unexpected alerts | Handles `UnexpectedAlertPresentException` |

## 📋 Supported External Link Tag Extraction

| Tag | Description | Example |
|-----|-------------|---------|
| `<a>` | Hyperlink | `<a href="http://evil.com">` |
| `<link>` | CSS import | `<link href="http://evil.com/evil.css">` |
| `<iframe>` `<frame>` | Inline frame | `<iframe src="http://evil.com">` |
| `<img>` | Image / Tracking pixel | `<img src="http://evil.com/pixel.gif">` |
| `<script>` | External script | `<script src="http://evil.com/a.js">` |
| `<embed>` `<object>` | Plugin content | `<embed src="http://evil.com/file.swf">` |
| `<meta refresh>` | Auto redirect | `<meta http-equiv="refresh" content="0;url=http://evil.com">` |
| `<style>` / `style` attribute | Background image | `style="background:url(http://evil.com/bg.jpg)"` |
| `<form>` | Form submission | `<form action="http://evil.com/submit">` |
| `<area>` | Image map | `<area href="http://evil.com/link">` |
| `<base>` | Base URL | `<base href="http://evil.com/">` |
| `<!-- -->` comments | Hidden links in comments | `<!-- http://evil.com/hidden -->` |
| `<audio>` `<video>` | Media resources | `<audio src="http://evil.com/audio.mp3">` |
| `<source>` | Media source | `<source src="http://evil.com/video.mp4">` |
| Inline `<script>` | URLs in scripts | `<script>var url="http://evil.com/api"</script>` |
| **Event Attributes** | onclick/onload events | `<div onclick="open('http://evil.com')">` |
| **Dynamic Injection** | document.write/innerHTML | `<script>document.write('<iframe src="evil.com">')</script>` |
| **Location Redirect** | window.location hijacking | `<script>window.location.href='http://evil.com'</script>` |

## 🚀 Quick Start

### Requirements

- Python 3.6+
- pip
- Chrome Browser (required for Selenium dynamic rendering)
- ChromeDriver (auto-downloaded by webdriver-manager)

### Install Dependencies

```bash
pip install requests beautifulsoup4 selenium webdriver-manager
```

### Run

```bash
python scanJS.py
```

## Configuration

Key configuration options in `ScannerConfig` (modify in code):

| Parameter | Default | Description |
|-----------|---------|-------------|
| `use_selenium` | `True` | Enable Selenium dynamic rendering |
| `headless_mode` | `True` | Run Chrome in headless mode |
| `page_load_timeout` | `10` | Page load timeout (seconds) |
| `dynamic_wait_time` | `3` | Wait time for dynamic content (seconds) |
| `download_timeout` | `5` | JS download timeout (seconds) |
| `request_timeout` | `5` | HTTP request timeout (seconds) |
| `max_file_size` | `0` | Max file size (0 = unlimited) |
| `allow_mixed_content` | `True` | Allow HTTPS pages to load HTTP resources |
| `extract_js_from_source` | `True` | Extract JS URLs from page source using regex |
| `connection_check_timeout` | `5` | Connection check timeout (seconds) |
| `download_js_extensions` | `('.js', '.mjs', '.cjs')` | JavaScript file extensions to download |

### Mixed Content Warnings
```bash
If you see (blocked:mixed-content) warnings, the scanner will still attempt to download these resources. You can enable allow_mixed_content in the configuration.
```

### Unexpected Browser Alerts
```bash
Unexpected Browser Alerts
The scanner automatically handles unexpected alerts (e.g., Baidu Maps license warnings). If you encounter other alert types, they will be dismissed automatically.
```

## How Pattern Deduplication Works

The scanner uses intelligent pattern detection to avoid scanning duplicate pages:

| URL Pattern | Example URLs | Scanned |
|-------------|--------------|---------|
| `/products/{id}` | `/products/1`, `/products/2`, `/products/3` | Only 1 |
| `/catalogue/{id}` | `/catalogue/product_123`, `/catalogue/product_456` | Only 1 |
| `/news/{id}/` | `/news/340/`, `/news/336/`, `/news/235/` | Only 1 |
| `/about-us` | `/about-us` | 1 (unique) |
| `/contact-us/` | `/contact-us/` | 1 (unique) |

**Result**: 156 discovered URLs → only 23 actually scanned (85% efficiency improvement)
